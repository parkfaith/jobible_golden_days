# Phase 2 기획서 — 카테고리 홈 & 절기 콘텐츠

> 작성일: 2026-02-16
> 상태: **검토 대기**

---

## 1. 배경 및 목적

### 현재 상태 (Phase 1)
- 앱 진입 → 바로 "오늘의 5개 카드" 표시 (좌우 스와이프)
- 카테고리 구분 없이 성경+명언+속담이 섞여서 5개 노출
- 날짜별 콘텐츠만 존재, 절기/계절 개념 없음
- 콘텐츠: 성경 26개, 명언 16개, 속담 8개, 시/글귀 0개 = 총 50개

### 사용자 관찰 (아버지 사용 패턴)
- 최근 콘텐츠를 **이것저것 둘러보시다가** 마음에 드는 것 발견 시 **카톡 공유**
- 움짤(GIF)도 좋아하심
- 설날 같은 **절기에 맞는 콘텐츠**가 나오면 공유 의향 높음
- 복잡한 메뉴보다 **"딱 맞는 콘텐츠"가 바로 보이는 것**을 선호

### 목표
1. 홈 화면에서 **카테고리별 탐색** 가능하게
2. **절기/계절에 맞는 콘텐츠**를 자동으로 상단 노출
3. 70대 어르신 기준 — **최소한의 탭으로 원하는 콘텐츠 도달**
4. 움짤(GIF) 콘텐츠 지원

## 2. 홈 화면 구조

### 2.1 레이아웃 (스크롤 가능한 단일 페이지)

```
┌─────────────────────────────┐
│  헤더 (로고 + 설정 버튼들)     │
├─────────────────────────────┤
│                             │
│  🎋 [절기 배너] (해당 시기만)  │  ← 설날, 추석 등 절기 콘텐츠 (자동 노출)
│  "설날 특별 콘텐츠 5선"        │
│                             │
├─────────────────────────────┤
│                             │
│  ✨ 오늘의 이야기              │  ← 매일 랜덤 5개 (기존 로직 유지)
│  [카드 미리보기 가로 스크롤]    │
│                             │
├─────────────────────────────┤
│                             │
│  📂 카테고리                  │
│  ┌──────┐ ┌──────┐          │
│  │ 말씀  │ │ 명언  │          │  ← 2열 그리드, 큰 터치 영역
│  └──────┘ └──────┘          │
│  ┌──────┐ ┌──────┐          │
│  │ 속담  │ │ 시   │          │
│  └──────┘ └──────┘          │
│  ┌──────┐ ┌──────┐          │
│  │ 좋은글 │ │ 움짤  │          │
│  └──────┘ └──────┘          │
│                             │
├─────────────────────────────┤
│  ❤️ 내가 저장한 글귀           │  ← 즐겨찾기 바로가기 (기존 모달 → 섹션)
│  [가로 스크롤 미리보기]        │
│                             │
└─────────────────────────────┘
```

### 2.2 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **최소 뎁스** | 홈 → 카드 뷰 (2단계, 최대). 서브메뉴 없음 |
| **큰 터치 영역** | 카테고리 버튼 최소 80x80px, 카드 미리보기 최소 160x200px |
| **자동 맥락** | 절기 콘텐츠는 시스템이 날짜 기반으로 자동 판단, 사용자 조작 불필요 |
| **점진적 노출** | 콘텐츠 없는 카테고리(시, 글귀)는 홈에서 숨김 처리 |
| **스크롤 vs 탭** | 탭 방식이 아닌 **세로 스크롤** 방식 — 어르신에게 가장 익숙한 패턴 |

## 3. 절기 콘텐츠 시스템

### 3.1 지원할 절기 목록 (1차)

절기를 너무 많이 만들면 관리 부담이 크므로, **공유 빈도가 높을 절기만** 선별합니다.

| 절기 | 시기 | 노출 기간 | 콘텐츠 예시 |
|------|------|-----------|------------|
| **설날** | 음력 1/1 (매년 변동) | 전후 3일 | 새해 덕담, 감사 말씀, 세배 인사 |
| **추석** | 음력 8/15 (매년 변동) | 전후 3일 | 추수 감사, 한가위 인사, 가족 명언 |
| **어버이날** | 5월 8일 | 전후 3일 | 부모 감사, 효도 말씀 |
| **크리스마스** | 12월 25일 | 전후 5일 | 성탄 성경, 사랑 메시지 |
| **새해** | 1월 1일 | 1/1~1/3 | 새해 소망, 결심 명언 |
| **봄** | 3~5월 | 계절 전체 | 봄 관련 시, 새 출발 |
| **여름** | 6~8월 | 계절 전체 | 건강 관련, 시원한 이미지 |
| **가을** | 9~11월 | 계절 전체 | 단풍, 감사, 수확 |
| **겨울** | 12~2월 | 계절 전체 | 따뜻함, 위로, 소망 |

> **음력 날짜 처리**: 설날·추석은 매년 양력 날짜가 달라지므로, 향후 3~5년치 양력 변환 날짜를 데이터에 미리 입력합니다 (외부 API 불필요, 오프라인 지원).

### 3.2 절기 콘텐츠 우선순위

```
명절(설날/추석) > 기념일(어버이날/크리스마스/새해) > 계절
```

- 명절이 겹치면 명절 우선
- 계절은 다른 절기가 없을 때의 **기본값** 역할
- 절기 배너에는 **명절/기념일만** 표시, 계절은 배너 없이 콘텐츠에 태그만 부여

## 4. 카테고리 시스템

### 4.1 카테고리 정의

| 카테고리 | key | 아이콘 | 현재 콘텐츠 수 | 비고 |
|----------|-----|--------|---------------|------|
| 성경 말씀 | `bible` | 📖 | 26개 | 기존 유지 |
| 명언 | `quote` | 💬 | 16개 | 기존 유지 |
| 속담 | `proverb` | 🏮 | 8개 | 기존 유지 |
| 시 | `poem` | 🌸 | 0개 → **확충 필요** | 김소월, 윤동주 등 저작권 만료 |
| 좋은 글 | `writing` | ✍️ | 0개 → **확충 필요** | 감동적인 짧은 글 |
| 움짤 | `gif` | 🎬 | 0개 → **신규** | GIF 애니메이션 |

> **콘텐츠 0개인 카테고리는 홈에서 숨김** — 데이터가 추가되면 자동 노출

### 4.2 카테고리 진입 시 UI

카테고리 탭 시 → **기존 카드 뷰어**(전체 화면 + 좌우 스와이프)로 전환
해당 카테고리의 콘텐츠를 날짜 기반 셔플로 표시

```
[홈 화면]  →  탭 "말씀"  →  [말씀 카드 뷰어] (전체 화면, 좌우 스와이프)
                              ← 뒤로가기 버튼으로 홈 복귀
```

### 4.3 움짤(GIF) 지원

| 항목 | 설계 |
|------|------|
| 저장 위치 | `public/images/gif/` |
| 파일 형식 | `.gif` (향후 `.webp` 애니메이션 가능) |
| 데이터 구조 | 기존과 동일 (`id`, `quote`, `author`, `bgImage`) + `type: "gif"` |
| 카드 렌더링 | `<img>` 태그로 GIF 재생 (자동, 루프) |
| 공유 | GIF 파일 직접 공유 (Web Share API files) — 캡처 불필요 |
| 용량 주의 | GIF 1장당 최대 2MB 권장, 총 20장 이내 시작 |

```jsonc
// gifs.json
[
  {
    "id": "g-01",
    "quote": "",
    "author": "",
    "source": "움짤",
    "bgImage": "/images/gif/flowers-01.gif",
    "type": "gif",
    "tags": ["꽃", "봄", "아름다운"]
  }
]
```

> **움짤은 글귀 없이 이미지만일 수도 있음** — `quote`가 빈 문자열이면 텍스트 오버레이 생략

## 5. 콘텐츠 신선도 전략 (Content Freshness)

### 5.1 문제 인식

| 현상 | 설명 |
|------|------|
| **콘텐츠 소진** | 현재 50개 ÷ 하루 5개 = **10일이면 전체 1회전**. 2주 지나면 같은 글이 반복 |
| **망각 곡선** | 3개월 이상 지난 콘텐츠는 대부분 기억하지 못함 → **다시 보면 새롭게 느낌** |
| **사용자 패턴** | 최근 것을 훑어보다 마음에 드는 것을 공유 → **"최근" 느낌이 중요** |

### 5.2 핵심 아이디어: "다시 만나는 오늘의 글귀"

**오래된 콘텐츠를 "재발견"이라는 이름으로 자연스럽게 다시 보여주기**

```
┌─────────────────────────────────────────┐
│                                         │
│  💎 다시 만나는 글귀                       │
│  "3개월 전 이맘때의 영감"                  │
│                                         │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ bg-img  │ │ bg-img  │ │ bg-img  │   │
│  │         │ │         │ │         │   │
│  │ "명언..." │ │ "말씀..." │ │ "속담..." │   │
│  └─────────┘ └─────────┘ └─────────┘   │
│                                         │
└─────────────────────────────────────────┘
```

- 홈 화면의 "오늘의 이야기" 아래에 배치
- **90일 전(약 3개월)** 날짜의 콘텐츠 5개를 자동 계산
- 기존 `dailyCurator.js`의 결정론적 알고리즘 활용 (추가 저장 불필요)
  - `getDailyContent(오늘 - 90일)` → 바로 계산 가능
- 앱 사용 시작 후 90일이 지나야 의미 있으므로, **90일 미만이면 숨김**

### 5.3 "오늘의 이야기" 선택 알고리즘 개선

현재 알고리즘은 날짜 시드만으로 셔플하기 때문에 콘텐츠 풀이 작으면 반복이 빠릅니다.
**최근 N일 내 노출된 콘텐츠를 피하는 로직**을 추가합니다.

```
[기존] 날짜 시드 → 셔플 → 앞 5개 선택

[개선] 날짜 시드 → 셔플 → 최근 7일 내 노출된 ID 제외 → 앞 5개 선택
                              ↑
                     결정론적 계산 (localStorage 불필요)
                     : 오늘-1 ~ 오늘-7의 결과를 미리 계산해서 제외 목록 생성
```

- **최근 7일 제외**: 50개 풀에서 7일×5개=35개 제외 → 남은 15개 중 5개 선택
- 콘텐츠 풀이 커지면(100개+) 자연스럽게 겹침이 줄어듦
- **완전 결정론적** — 같은 날짜는 항상 같은 결과, localStorage 불필요

### 5.4 콘텐츠 "세대" 관리

콘텐츠가 점진적으로 늘어날 것을 감안하여, 데이터에 **등록 시점**을 기록합니다.

```jsonc
{
  "id": 51,
  "quote": "새로운 명언...",
  "author": "저자",
  "bgImage": "/images/bg-51.jpg",
  "addedDate": "2026-03"   // 추가된 월 (연-월만)
}
```

활용 방법:

| 용도 | 설명 |
|------|------|
| **"이달의 새 글귀" 뱃지** | 최근 1개월 내 추가된 콘텐츠에 🆕 표시 |
| **신규 콘텐츠 우선** | 오늘의 이야기에서 새 콘텐츠를 1~2개 우선 배치 |
| **재발견 대상 선별** | `addedDate`가 3개월 이상 전인 콘텐츠만 "다시 만나는 글귀"에 노출 |

### 5.5 향후 콘텐츠 확충 로드맵

적은 콘텐츠 풀에서 반복감을 줄이려면, 결국 **콘텐츠 양 자체를 늘리는 것**이 본질적인 해결입니다.

| 시기 | 목표 | 효과 |
|------|------|------|
| Phase 2 출시 시 | 100개+ (현재 50 → 2배) | 반복 주기 20일 |
| 3개월 후 | 200개+ | 반복 주기 40일 (≈ 매일 새로움) |
| 6개월 후 | 300개+ | 사실상 반복 인지 불가 |

확충 우선순위:
1. **시(poems)** — 김소월, 윤동주, 한용운 등 저작권 만료 시인 (확보 용이)
2. **좋은 글(writings)** — 짧은 수필, 감동 에세이 발췌
3. **절기 콘텐츠(seasonal)** — 명절/기념일별 5~10편
4. **명언(quotes) 추가** — 동양 고전, 성현의 말씀
5. **움짤(gifs)** — 꽃, 자연, 계절 GIF

### 5.6 홈 화면 최종 레이아웃 (재발견 섹션 포함)

```
┌─────────────────────────────┐
│  헤더                        │
├─────────────────────────────┤
│  🎋 [절기 배너] (해당 시기만)  │
├─────────────────────────────┤
│  ✨ 오늘의 이야기 (5개)        │  ← 매일 새로운 5개
│  [가로 스크롤 미리보기]        │
├─────────────────────────────┤
│  💎 다시 만나는 글귀 (5개)     │  ← 90일 전 콘텐츠 (사용 90일 후 노출)
│  [가로 스크롤 미리보기]        │
├─────────────────────────────┤
│  📂 카테고리 (2열 그리드)      │
├─────────────────────────────┤
│  ❤️ 내가 저장한 글귀           │
└─────────────────────────────┘
```

## 6. 네비게이션 흐름

### 5.1 전체 흐름도

```
[홈 화면 (스크롤)]
  │
  ├─ [절기 배너 탭] ──→ [절기 카드 뷰어] ──→ 뒤로(홈)
  │
  ├─ [오늘의 이야기 탭] ──→ [기존 카드 뷰어] (변경 없음) ──→ 뒤로(홈)
  │
  ├─ [카테고리 탭] ──→ [카테고리별 카드 뷰어] ──→ 뒤로(홈)
  │
  └─ [즐겨찾기 탭] ──→ [즐겨찾기 카드 뷰어] ──→ 뒤로(홈)
```

### 5.2 뒤로가기

- Android 하드웨어 백 버튼 / 브라우저 뒤로가기 지원
- 카드 뷰어 좌상단에 **큰 뒤로가기 버튼** (← 아이콘, 48px+)
- `history.pushState` 활용하여 브라우저 뒤로가기 시 홈으로 복귀 (React Router 미사용, 직접 관리)

### 7.2 라우팅 전략

React Router를 사용하지 않는 현재 아키텍처를 유지합니다.

```jsx
// App.jsx — 상태 기반 화면 전환
const [view, setView] = useState('home'); // 'home' | 'viewer'
const [viewerConfig, setViewerConfig] = useState(null);

// viewerConfig 예시:
// { type: 'daily', contents: [...] }
// { type: 'category', category: 'bible', contents: [...] }
// { type: 'seasonal', season: '설날', contents: [...] }
// { type: 'favorites', contents: [...] }
```

- `view === 'home'` → `<Home />` 렌더링
- `view === 'viewer'` → `<CardViewer />` 렌더링
- `popstate` 이벤트로 브라우저 뒤로가기 처리

## 11. 고려사항 및 제약

### 11.1 콘텐츠 수급

| 항목 | 현황 | 필요 작업 |
|------|------|----------|
| 시 (poems) | 0개 | 저작권 만료 한국 시 10~20편 수급 필요 |
| 좋은 글 (writings) | 0개 | 짧은 감동 글귀 10~20편 수급 필요 |
| 움짤 (gifs) | 0개 | 저작권 프리 GIF 10~20개 수급 필요 |
| 절기 콘텐츠 (seasonal) | 0개 | 절기별 5~10편씩 수급 필요 |
| 절기 배경 이미지 | 0개 | 절기별 테마 이미지 수급 필요 |

### 11.2 PWA 용량

- 현재: 이미지 50장 약 5MB
- GIF 추가 시: GIF 1장당 1~2MB → 20장 시 최대 40MB 추가
- **대응**: GIF는 런타임 캐싱 (Workbox)으로 처리, precache에서 제외
- 정적 이미지는 기존처럼 precache 유지

### 11.3 오프라인 지원

- 홈 화면 + 기본 콘텐츠(성경/명언/속담): **완전 오프라인**
- GIF/절기 이미지: **첫 조회 후 캐싱** (오프라인 시 캐시 된 것만 표시)
- 캐시 미스 시: 플레이스홀더 이미지 표시

### 11.4 성능

- 홈 화면 이미지 미리보기: **lazy loading** + 작은 썸네일 사용
- GIF: 뷰포트 진입 시에만 재생 (Intersection Observer)
- 카테고리 그리드 배경: CSS `background-image`로 저해상도 처리
