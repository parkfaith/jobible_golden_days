# Project Rules

이 문서는 프로젝트 진행 시 반드시 준수해야 할 규칙들을 정의합니다.

## 1. Language (언어)

- 모든 응답은 **한국어(Korean)**로 작성합니다. (진행 상황 업데이트 포함)

## 2. Code Comments (코드 주석)

- 코드 내 주석은 반드시 **한국어**로 작성합니다.

## 3. Explanation (설명)

- 기술적인 용어는 영어를 병기합니다. (예: 변수(Variable), 함수(Function))
- 설명은 이해하기 쉽도록 한국어로 풀어서 작성합니다.

## 4. CHANGELOG.md (변경 이력 관리)

- 코드 수정 후에는 반드시 `CHANGELOG.md` 파일을 업데이트합니다.
- 변경 이력에는 다음 내용을 포함해야 합니다:
  - **날짜 (Date)**: 변경 작업이 수행된 날짜
  - **카테고리 (Category)**: 변경 유형 (신규 기능, UI 개선, 버그 수정, 리팩토링 등)
  - **상세 내용 (Description)**: 변경된 작업에 대한 상세 설명
  - **수정 파일 목록 (Modified Files)**: 수정된 파일들의 목록

---

_이 규칙은 프로젝트가 시작될 때마다 적용됩니다._
