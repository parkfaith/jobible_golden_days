
[PRD] 시니어 정서 케어를 위한 일일 영감 큐레이션 웹앱
1. 제품 개요 (Product Overview)
목적: 70세 이상 시니어 어르신들이 매일 아침 새로운 명언, 배경 이미지, 배경 음악을 통해 정서적 위안과 영감을 얻을 수 있도록 돕는 서비스.
핵심 가치: 단순성(Simplicity), 가독성(Readability), 정서적 연결(Connection).
대상 사용자: 스마트폰 사용에는 익숙하나 복잡한 UI 및 작은 글씨에 피로감을 느끼는 70세 이상 어르신.

2. 주요 기능 (Key Features)
2.1. 일일 콘텐츠 자동 생성 및 로테이션
요구사항: 매일 정해진 시간에 5개의 새로운 콘텐츠(글귀 + 배경 이미지 + 음악) 세트가 생성되어야 함.
구현 방식:
JavaScript Date() 객체를 활용해 당일 날짜와 매칭되는 JSON 데이터를 호출.
GitHub Actions를 통해 외부 API(API Ninjas 등)에서 명언을 수집하여 자동 업데이트 환경 구축.
5개의 세트는 슬라이드나 간단한 버튼으로 전환 가능해야 함.
2.2. 관리자 수동 콘텐츠 생성
요구사항: 자동 생성 외에 관리자가 직접 특별한 날(명절, 기념일 등)에 맞춘 콘텐츠를 업로드할 수 있어야 함.
구현 방식: Contentful과 같은 Headless CMS를 연동하여 코드 수정 없이 텍스트와 이미지를 관리자 화면에서 업로드.
2.3. 시니어 특화 UI/UX (Accessibility)
가독성: 폰트 크기 29px(21.6pt) 이상 유지, 텍스트와 배경 간 명도 대비 7:1 이상 확보.
시각적 보조: 이미지 위에 rgba(0,0,0,0.3) 수준의 어두운 오버레이를 씌워 글자 시인성 극대화.
조작성: 모든 클릭 요소(버튼, 아이콘)는 48x48dp 이상의 크기를 확보하여 오클릭 방지.
2.4. 아카이브 및 공유 기능
달력 보기: React-Calendar를 활용해 날짜별로 과거 콘텐츠를 쉽게 찾아볼 수 있는 기능 제공.
간편 공유: Web Share API를 이용해 카카오톡, 문자 메시지로 즉시 전송.

3. 기술 스택 및 비용 최적화 (Tech Stack & Budget)
모든 기술 스택은 무료 티어(Free Tier) 활용을 원칙으로 합니다.

4. 설치 및 접근성 전략 (PWA)
홈 화면 추가: PWA 기술을 적용하여 별도의 앱스토어 설치 없이 스마트폰 홈 화면에 '바로가기' 아이콘 생성.
오프라인 모드: 서비스 워커(Service Worker)를 통해 네트워크가 불안정한 상황에서도 기존에 불러온 글귀 확인 가능.
독립 실행: 브라우저 주소창을 제거한 스탠드얼론(Standalone) 모드로 몰입감 제공.

5. 향후 로드맵 (Roadmap)
Phase 1: 핵심 기능 개발 (일일 5개 콘텐츠 자동 로테이션 및 PWA 설정).
Phase 2: 관리자 페이지(CMS) 연동 및 수동 업로드 기능 검증.
Phase 3: 시니어 사용자 대상 사용성 테스트(UT) 및 가독성 피드백 반영.
